## Architecture Snapshot

- **API prefix**: `/api/v1`
- **Global guards** (via `APP_GUARD`): `ThrottlerGuard` (300 req/min/IP) â†’ `JwtAuthGuard`. All routes auth-required by default; opt out with `@Public()`.
- **Global pipes**: `ValidationPipe({ whitelist: true, forbidNonWhitelisted: true })` â€” every DTO field must be decorated.
- **Global interceptors**: `RedisCacheInterceptor` (public-GET cache) â†’ `TransformInterceptor` (envelope `{ success, data, message }`).
- **Trust proxy**: `app.set('trust proxy', 2)` â€” CF â†’ Caddy â†’ Node. `req.ip` is the real client IP. Don't bypass.
- **Path alias**: `@/*` â†’ `./src/*`. **Module resolution `nodenext`** â€” local imports MUST end in `.js` (e.g. `import { foo } from './foo.js'`).

See `docs/system-architecture.md` for full pipeline, security baseline, caching layers.

## Key Decorators (`common/decorators/`)

| Decorator | Use |
|---|---|
| `@Public()` | Skip `JwtAuthGuard` |
| `@CurrentUser()` / `@CurrentUser('sub')` | Extract user (or specific JWT field) from request |
| `@Roles('admin')` | Role gate â€” JWT payload `role: 'admin' \| 'user'` |
| `@CacheTTL(seconds)` | TTL for `RedisCacheInterceptor` on a public GET |

JWT payload shape: `{ sub: number, uuid: string, email: string, role: 'admin' \| 'user' }`.

## Database (Drizzle)

- Driver: `postgres.js`. Inject DB via `@Inject(DRIZZLE)` with type `DrizzleDB` from `database/drizzle.provider.ts`.
- Schema files: `src/database/schema/*.schema.ts` â€” barrel-exported via `index.ts`. Relations in `relations.ts`.
- Migrations: `src/database/migrations/`, generated by `drizzle-kit generate`.
- **Concurrent indexes** (e.g. `CREATE INDEX CONCURRENTLY`) MUST live in their own no-transaction migration (see migration `0021` for the pattern).
- Generated columns: stock `unaccent()` is STABLE â€” wrap in IMMUTABLE SQL fn (`normalize_title`) to use in `GENERATED ALWAYS AS ... STORED`.
- Seeds: `src/database/seed/`, run via `pnpm run db:seed`.

## Caching (3 Layers)

```
HTTP req â†’ [L1 RedisCacheInterceptor] â†’ [L2 service-level Redis] â†’ [L3 in-memory Map] â†’ DB
```

- **L1** â€” public GET only, skipped for authenticated users. Key `cache:<url>`. Set per-route TTL with `@CacheTTL(seconds)`.
- **L2** â€” service-level (rankings, taxonomy). Hand-rolled in services.
- **L3** â€” `TaxonomyService` `Map` (10min, 100 entries) â€” benefits authed users that bypass L1.

**Default TTLs**: manga 180s Â· ranking 300s (HTTP) / 900s (service) Â· search 120s Â· taxonomy 600s Â· stats 3600s.

**Invalidation**:
- Event-driven: emit `chapter.created` / `manga.updated` â†’ targeted `DEL`.
- Cron: `rankings:*` flushed every 5 min.
- Counter-flush: view buffers â†’ DB every 5 min (`CounterFlushJob`, gated by `redisStatus.available`).

## Redis Optional / Graceful Degradation

Inject `REDIS_AVAILABLE` token (`RedisStatus`) wherever fallback behavior matters. `stubRedisClient()` provides no-op `get/set/del/incr/expire/scan/getdel/...` when Redis unreachable.

| Path | Redis up | Redis down |
|---|---|---|
| View tracking | Buffer in Redis, flushed by cron | Direct `views + 1` SQL increment |
| Refresh tokens | Redis + DB dual-write | DB-only via `refresh_tokens` table |
| Login lockout | Per-email counter (sha256 key) | Warn + per-IP throttle only |
| OAuth code exchange | 60s TTL, single-use `GETDEL` | OAuth login temporarily unavailable |

## Auth Invariants (don't break these)

- **OAuth**: callback returns one-time `code` in query string. Frontend POSTs `/auth/google/exchange` to swap â†’ tokens. Never put tokens in URL fragment (legacy fragment behind `OAUTH_LEGACY_FRAGMENT=1` only).
- **Refresh family**: each refresh carries `jti` + `family_id`. Replay of revoked `jti` â†’ revoke whole family + audit log `security.refresh-reuse`.
- **Silent Google linking blocked**: incoming Google identity with no `googleId` match but matching email â†’ 409 `ACCOUNT_EXISTS_NO_LINK`.
- **JWT secret fail-fast**: `JWT_ACCESS_SECRET` / `JWT_REFRESH_SECRET` must be â‰Ą32 bytes outside test env. Missing/short â†’ process exits at boot.
- **Content-rating filter â‰  access control**: `NSFW_RATINGS` is a *discovery* preference. Apply to list/discovery endpoints. NEVER apply to direct-intent detail lookups (`GET /manga/:slug`).

## SSRF â€” Always Use `safe-http`

Any code that fetches an upstream-controlled URL (import adapters, image mirroring) MUST go through `common/utils/safe-http.util.ts`:
- HTTPS only Â· DNS pre-resolution rejecting private/loopback/link-local Â· `redirect: 'manual'` with re-validation per hop Â· max 3 hops Â· 30s timeout Â· 20MB body cap.

## Recipes

### Add a public cached GET endpoint
1. Controller method: `@Get(...)` + `@Public()` + `@CacheTTL(180)`.
2. If the response depends on data mutated elsewhere, emit an invalidation event (`chapter.created`, `manga.updated`) from the mutating service and add a listener in `JobsModule` that `DEL`s the relevant cache keys.
3. DTO with `class-validator` decorators (validation pipe is global).
4. Add unit test for the service + integration test for the route.

### Add a new module
1. `nest g module <name>` (or hand-roll). Place under `src/modules/<name>/`.
2. Import in `app.module.ts`.
3. Co-locate `*.controller.ts`, `*.service.ts`, `dto/`, `*.spec.ts`.
4. If the module needs DB: inject `@Inject(DRIZZLE) db: DrizzleDB`, add schema file under `database/schema/`, export via barrel.

### Add a Drizzle migration
1. Edit schema file under `src/database/schema/`.
2. `pnpm run db:generate` â†’ review generated SQL in `migrations/`.
3. For `CREATE INDEX CONCURRENTLY`: split into a separate migration file with no-transaction directive.
4. `pnpm run db:migrate` to apply locally.
5. Update `relations.ts` if the change adds/changes FK relationships.

### Add a Redis-dependent feature with graceful fallback
1. Inject `@Inject(REDIS_AVAILABLE) redisStatus: RedisStatus`.
2. Branch on `redisStatus.available` â€” implement DB fallback for the down path.
3. Test both branches (toggle `REDIS_AVAILABLE` provider in test module).

## Forbidden Patterns

- **No silent Redis dependence** â€” every Redis path needs a `redisStatus.available` check or must clearly document "Redis required" and fail loud.
- **No bypassing `safe-http`** for upstream-controlled URLs.
- **No applying `NSFW_RATINGS` to detail/share/embed routes** â€” preference, not gate.
- **No `origin: true`** in CORS config â€” explicit allowlist only.
- **No `.js`-less local imports** â€” `nodenext` enforces `.js` extension.
- **No new `enhanced-*.ts` / `*-v2.ts` files** â€” modify the existing module in place.
- **No committing without `pnpm run lint` + `pnpm run test`** passing.
